# Signal Freshness Checklist

Before writing any AI×GRC×FSI Signals post, verify each signal is fresh. The user explicitly rejects dated signals.

## Global Signals
- [ ] IMF blog (imf.org/en/Blogs) — check current month
- [ ] EU AI Act updates (European Commission AI Office, euronews)
- [ ] IOSCO/FSB pronouncements
- [ ] G7 AI governance communiques
- [ ] NIST AI RMF updates

## Australia Signals
- [ ] APRA System Risk Outlook (apra.gov.au/news-and-publications)
- [ ] APRA media releases
- [ ] ASIC newsroom (asic.gov.au/newsroom)
- [ ] ASIC Chair speeches (Tech Council of Australia panels, Parliamentary committees)
- [ ] RBA pronouncements
- [ ] OAIC guidance on AI

## AWS Signals
- [ ] AWS ML blog (aws.amazon.com/blogs/machine-learning/) — current month
- [ ] AWS Security Agent / DevOps Agent updates
- [ ] Bedrock AgentCore releases
- [ ] Amazon Q updates
- [ ] AWS re:Invent announcements (if within 2 weeks)
- [ ] AWS Marketplace AI listings

## Red Flags — Reject if:
- Signal is from >3 months ago
- No specific date or source attached
- The development is just a re-announcement of an older policy

## Green Flags — Accept if:
- Within the current month
- A new regulation/standard just came into force (even if announced earlier)
- A new enforcement action or penalty was issued
- A new AWS product/feature went GA or preview

## Sources in Comment
Always verify the URL resolves before including it in the sources comment.
